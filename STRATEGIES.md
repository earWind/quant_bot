# 常用量化交易策略

本文整理常见量化交易策略类型，方便后续在项目中扩展、回测和对比。

## 1. 趋势跟踪策略

趋势跟踪策略认为价格一旦形成趋势，往往会延续一段时间，因此选择顺势交易。

常见方法：

- 均线交叉：短期均线上穿长期均线买入，下穿卖出。
- 突破策略：价格突破近期高点买入，跌破近期低点卖出。
- MACD 策略：利用快慢线和柱状图判断趋势变化。
- ADX 策略：用趋势强度指标过滤震荡行情。

适合行情：

- 单边上涨或单边下跌行情。
- 波动方向较明确的市场。

主要风险：

- 震荡行情中容易频繁假突破。
- 信号通常有滞后性，可能追高或杀跌。

## 2. 均值回归策略

均值回归策略认为价格偏离合理区间后，有较大概率回到平均水平。

常见方法：

- 布林带：价格跌破下轨买入，回到中轨或上轨卖出。
- RSI：RSI 过低视为超卖，RSI 过高视为超买。
- Z-score：衡量价格或价差偏离均值的程度。
- 价差回归：交易两个相关资产之间的价差偏离。

适合行情：

- 横盘震荡行情。
- 有稳定均值或稳定价差关系的市场。

主要风险：

- 单边趋势中可能越跌越买或越涨越空。
- 均值关系可能失效。

## 3. 套利策略

套利策略利用不同市场、不同品种或不同合约之间的价格差异获利。

常见类型：

- 跨交易所套利：同一资产在不同交易所价格不同。
- 期现套利：现货和期货之间存在异常价差。
- 三角套利：利用三个交易对之间的汇率偏差。
- 统计套利：利用历史相关关系或价差关系进行交易。

适合行情：

- 市场分割明显、流动性充足的场景。
- 手续费、滑点和延迟可控的场景。

主要风险：

- 手续费和滑点可能吃掉利润。
- 下单延迟可能导致价差消失。
- 交易所提现、风控、接口异常会影响执行。

## 4. 网格交易策略

网格交易策略在预设价格区间内按固定间隔挂买卖单，低买高卖，反复赚取波动收益。

常见方法：

- 等差网格：每个网格价格间距固定。
- 等比网格：每个网格按固定百分比间隔。
- 动态网格：根据波动率或趋势调整网格区间。

适合行情：

- 横盘震荡行情。
- 价格在一定区间内反复波动的市场。

主要风险：

- 单边下跌会积累大量亏损仓位。
- 单边上涨可能提前卖飞。
- 网格区间设置不合理会影响收益和风险。

## 5. 做市策略

做市策略通过同时挂买单和卖单，为市场提供流动性，并赚取买卖价差。

常见方法：

- 固定价差做市：在中间价两侧挂固定价差订单。
- 库存控制做市：根据当前持仓调整买卖报价。
- 订单簿做市：根据盘口深度、成交量和买卖压力动态报价。

适合行情：

- 流动性较好、价差稳定的市场。
- 接口速度和撤单能力较强的环境。

主要风险：

- 行情突然单边波动时容易产生库存亏损。
- 高频撤单和挂单对系统稳定性要求较高。
- 手续费等级和撮合优先级影响很大。

## 6. 动量策略

动量策略认为近期表现强的资产未来一段时间可能继续强，近期表现弱的资产可能继续弱。

常见方法：

- 时间序列动量：资产自身近期涨幅较强则买入。
- 横截面动量：在多个资产中买入强势资产，卖出弱势资产。
- 相对强弱排序：按收益率、成交量或波动调整收益进行排名。

适合行情：

- 多资产组合交易。
- 市场存在明显强弱分化时。

主要风险：

- 行情快速反转时回撤较大。
- 过度追逐短期涨幅可能买在高位。

## 7. 多因子策略

多因子策略综合多个指标或特征，对资产进行打分、排序，并构建交易组合。

常见因子：

- 动量因子：近期收益、相对强弱。
- 波动率因子：历史波动率、真实波幅。
- 成交量因子：成交量变化、换手率。
- 价值因子：估值、基本面指标，常见于股票市场。
- 质量因子：盈利质量、财务稳定性，常见于股票市场。

适合行情：

- 股票、ETF、多币种或多品种组合。
- 有较丰富历史数据和可解释特征的场景。

主要风险：

- 因子可能阶段性失效。
- 因子之间可能高度相关，导致组合并不分散。
- 回测中容易出现过拟合。

## 8. 机器学习策略

机器学习策略使用模型从历史数据中学习规律，用于预测价格方向、收益率、波动率或交易信号。

常见模型：

- 逻辑回归、线性回归。
- 随机森林、XGBoost、LightGBM。
- LSTM、Transformer 等深度学习模型。
- 聚类、异常检测等无监督模型。

适合行情：

- 数据量充足、特征较丰富的场景。
- 有明确训练、验证和样本外测试流程的策略研究。

主要风险：

- 过拟合。
- 数据泄漏。
- 样本外表现不稳定。
- 模型可解释性较弱。

## 9. 高频交易策略

高频交易策略在极短时间尺度上捕捉微小价格变化，通常依赖低延迟系统和订单簿数据。

常见方法：

- 盘口不平衡策略。
- 微结构预测策略。
- 延迟套利。
- 高频做市。

适合行情：

- 流动性极高、撮合速度快的市场。
- 有低延迟基础设施和稳定交易接口的团队。

主要风险：

- 基础设施成本高。
- 普通网络环境很难获得优势。
- 接口限制、撮合规则和手续费对结果影响很大。

## 10. 风险平价与组合再平衡策略

这类策略不一定直接预测涨跌，而是根据风险分配资金，并定期调整资产权重。

常见方法：

- 固定比例再平衡：例如 BTC、ETH、现金按固定比例配置。
- 风险平价：让不同资产对组合风险的贡献尽量接近。
- 波动率目标：根据市场波动率调整总仓位。

适合行情：

- 中长期资产配置。
- 多资产组合管理。

主要风险：

- 极端行情下不同资产相关性可能同时升高。
- 再平衡频率过高会增加交易成本。
- 风险估计依赖历史数据，可能滞后。

## 当前项目可优先扩展的方向

当前项目已有基于短期均线和长期均线的趋势跟踪思路。后续可以优先尝试：

1. 完善均线趋势策略的止损、止盈和仓位管理。
2. 增加布林带或 RSI 均值回归策略，与均线策略对比。
3. 增加网格交易策略，适配震荡行情。
4. 扩展回测模块，统一比较收益率、最大回撤、胜率和交易次数。
5. 增加策略参数配置，让不同策略可以在 Web 面板中切换。

## 策略开发注意事项

- 所有策略上线前都应先做历史回测和模拟盘验证。
- 回测需要计入手续费、滑点和资金费率等真实交易成本。
- 不要只看收益率，也要关注最大回撤、胜率、盈亏比和持仓时间。
- 避免过度优化参数，否则容易出现样本内表现很好、实盘表现很差的问题。
- 实盘前应确认 API 权限、仓位限制、风控参数和异常处理逻辑。
